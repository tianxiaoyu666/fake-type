## 功能特性

### 假装打代码
- 预先准备好代码内容
- 在目标文件中随便敲键盘，每按一个键显示一个预设字符
- 支持多文件独立映射
- 回车、Tab 等特殊键正常工作
- 退格键同步回退（万一演砸了还能补救）

### 假终端输出
- 自定义终端前缀（完美模拟 PowerShell/Bash）
- 预设终端命令和输出结果
- 乱敲键盘逐字符显示预设内容
- 评委根本看不出来

### 数据持久化
- 映射关系自动保存
- 重启编辑器后自动恢复
- 进度实时保存（中途关机也不怕）

### 完全静默模式
- **零提示、零弹窗** —— 所有操作完全静默
- 切换状态没有任何通知
- 评委凑近看也发现不了异常
- 就算站你背后也看不出来

### 一键暂停/恢复
- 快捷键秒切换（默认 `Ctrl+F2`）
- 侧边栏状态一目了然
- 万一评委要凑近看？一秒暂停，若无其事

### 智能补全同步（高级功能）
- 暂停后可以正常使用 VS Code 的代码补全
- 恢复后**自动检测已输入的内容**
- **跳过已输入部分**，继续假打字
- 不会重复输出，衔接天衣无缝

**使用场景**：演示时偶尔用一下补全，看起来更真实。

```
1. Ctrl+F2 暂停
2. 输入 "con"，弹出补全
3. Tab 接受补全 -> "console"
4. Ctrl+F2 恢复
5. 插件自动跳过 "console"，继续输出后面的内容
```

**评委心想**：这小伙子 IDE 用得挺溜啊，还知道用补全。

## 安装方式

### 方式一：从 VSIX 安装（推荐）
1. 下载 `.vsix` 文件
2. 在 VS Code/Cursor 中按 `Ctrl+Shift+X` 打开扩展面板
3. 点击右上角 `...` -> "从 VSIX 安装..."
4. 选择下载的文件
5. 开始你的表演

### 方式二：从源码构建（适合想装逼的）

```bash
git clone https://github.com/tianxiaoyu666/fake-type.git
cd fake-type
npm install
npm run compile
npx @vscode/vsce package --allow-missing-repository --allow-star-activation
```

## 使用方法

### 代码映射

1. 点击左侧活动栏的 **Fake Type** 图标
2. 在"代码映射"区域：
   - 点击"选择"按钮选择目标文件
   - 粘贴你要"表演"的代码
   - 点击"保存"
3. 打开目标文件，随便敲键盘
4. 看着完美的代码一个字一个字出现
5. 评委：这小伙子代码写得真熟练啊

### 终端映射

1. 在"终端命令"区域：
   - 设置终端前缀（如 `PS C:\Users\xxx> `）
   - 粘贴完整的终端内容（包括命令和输出）
   - 点击"添加"
2. 点击"打开演示终端"
3. 随便敲键盘，看着命令和输出完美呈现
4. 评委：这小伙子对命令行很熟练啊

### 紧急暂停

评委突然凑过来了？别慌！

1. 按 `Ctrl+F2`（或你自定义的快捷键）
2. 立刻恢复正常输入模式
3. 淡定地敲几行真代码（反正你也背过）
4. 评委走了，再按一下恢复
5. 插件会自动同步你刚才输入的内容，继续假打字

**全程无任何提示弹出，神不知鬼不觉。**

## 快捷键

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+F2` | 暂停/恢复 | 一键切换，完全静默，无任何提示 |

### 自定义快捷键

默认的 `Ctrl+F2` 不顺手？没问题，随便改：

**方法一（推荐）**：点击侧边栏顶部的"修改"链接，自动打开设置页面

**方法二**：手动打开
1. 按 `Ctrl+K` 然后按 `Ctrl+S` 打开键盘快捷方式
2. 搜索 `fakeType.toggle`
3. 双击，按下你想要的快捷键
4. 完成

**建议**：选一个不容易误触、又方便按的键，比如：
- `Ctrl+Shift+F2`
- `Alt+F2`
- `Ctrl+反引号`

注意：避免和 VS Code 内置快捷键冲突，否则可能出问题。

## 快捷命令

按 `Ctrl+Shift+P` 打开命令面板，输入 `Fake Type`：

| 命令 | 说明 |
|------|------|
| `Fake Type: 切换启用/禁用` | 暂停/恢复（推荐用快捷键） |
| `Fake Type: 从剪贴板加载` | 快速为当前文件加载预设内容 |
| `Fake Type: 重置到开头` | 重置当前文件进度（再演一遍） |
| `Fake Type: 打开演示终端` | 打开伪终端 |

## 界面说明

侧边栏顶部有一个状态控制栏：

```
+----------------------------------+
| [绿点] 已启用        [暂停按钮]  |
|       快捷键: Ctrl+F2 (修改)     |
+----------------------------------+
```

- **绿点** = 已启用，正在假打字
- **红点** = 已暂停，正常输入模式
- 点击按钮或按快捷键切换

状态栏（右下角）也会显示：
- `未映射` = 当前文件没有预设内容
- `已暂停` = 功能已暂停
- `xxx 字符` = 剩余多少字符待输出

## 常见问题

**Q: 这不是作弊吗？**

A: 你管提前拿题、反复排练、背诵代码叫什么？大家都在演，我只是演得更专业。

**Q: 被发现了怎么办？**

A: 放心，评委不会看你的插件列表的。再说了，他们可能也知道是怎么回事，只是不说而已。而且现在完全静默，没有任何提示弹出，发现不了的。

**Q: 这对真正学技术的人公平吗？**

A: 兄弟，你觉得现在的比赛对他们公平吗？

**Q: 暂停后输入的内容，恢复后会重复吗？**

A: 不会。恢复时插件会自动检测你已经输入了多少，从正确的位置继续。这就是"智能同步"功能。

**Q: 快捷键冲突了怎么办？**

A: 点侧边栏的"修改"链接，或者手动去键盘快捷方式里改成别的就行。

**Q: 有没有提示告诉我切换成功了？**

A: 没有，故意的。完全静默，评委看不到任何异常。你可以看侧边栏的状态指示灯，或者状态栏的文字。
